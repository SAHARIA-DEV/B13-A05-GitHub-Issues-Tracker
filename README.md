## Assignment Questions

### 1. What was the most difficult part of the assignment and how did you solve it?

The most difficult part of the assignment was fetching the data from the API and displaying the issues dynamically on the page. At first, I had some trouble understanding the structure of the API response and how to properly show each issue as a card in the UI.

To solve this, I used console.log() to inspect the API response and understand the data structure. After that, I created a separate function to render the issues and used JavaScript methods like filter() to separate open and closed issues. Breaking the problem into smaller steps helped me implement the feature correctly.

### 2. If you had more time, what improvements would you make?

If I had more time, I would improve a few things in the project.

First, the login system currently uses hardcoded credentials. It would be better to implement a real authentication system with a backend for better security.

Second, I would improve the UI by adding more interactive elements like hover effects or better styling for labels.

Finally, I would improve the search feature so that it supports real-time searching while the user types, instead of clicking a button.

### 3. What resources did you use while completing the assignment?

While working on this assignment, I used several resources to help me complete the project.

I referred to MDN documentation to better understand JavaScript functions like fetch(), filter(), and DOM manipulation. I also used the Tailwind CSS documentation for styling the UI.

In addition, I used Google search to find solutions for specific problems and examples that helped me understand certain concepts better.

### 4. What did you learn from this assignment?

From this assignment, I learned several practical concepts related to frontend development.

I learned how to fetch data from an API and use that data to dynamically generate content on a webpage. I also practiced working with JavaScript to manipulate the DOM and update the UI based on user interaction.

Another important thing I learned was how organizing code into smaller functions makes the project easier to manage and understand.

### 5. Which part of your solution are you most proud of?

The part I am most proud of is the issue filtering system with the All, Open, and Closed tabs.

This feature allows users to quickly switch between different issue statuses and view the relevant issues. It makes the application easier to use and similar to how issue tracking works on platforms like GitHub.

Implementing this feature helped me practice using JavaScript filtering and managing active states in the UI.
